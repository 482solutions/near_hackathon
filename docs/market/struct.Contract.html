Expand description

Struct for storing various information about market state

Fields

owner_id: AccountIdasks: UnorderedMap<ContractAndId, Ask>

To keep track of the ask, we map the ContractAndTokenId to a Sale. the ContractAndTokenId is the unique identifier for every sale. It is made up of the contract ID + DELIMITER + UUIDv4

bids: UnorderedMap<ContractAndId, Bid>

Same with bids

asks_id: u128

Current id for asks

bids_id: u128

Current id for bids

asks_by_owner_id: LookupMap<AccountId, UnorderedSet<ContractAndId>>

Keep track of all the Ask IDs for every account ID

bids_by_owner_id: LookupMap<AccountId, UnorderedSet<ContractAndId>>

Keep track of all the Bids IDs for every account ID

storage_deposits: LookupMap<AccountId, Balance>

Keep track of the storage that accounts have paid
